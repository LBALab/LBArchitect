Track Script: GOTO_SYM_POINT (command)

GOTO_SYM_POINT makes the Actor go to specified Point using current Animation, but facing back to the Point.

Syntax:
  • GOTO_SYM_POINT id(0 to 255 integer)
Parameters:
  • id is the index of the target Point. The Point must exist in the Scene.

This command uses Animation that has been assigned to the Actor for example with ANIM command, but unlike the GOTO_POINT command, GOTO_SYM_POINT will make the Actor turn back to the target Point (he will face the direction opposite to the target Point). If the Animation is a static animation (doesn't move the character), for example standing Animation, the Actor will not make a move. To make the Actor move it is necessary to assign a moving Animation to it (a walking back Animation for instance) and then point it back to desired direction with GOTO_SYM_POINT command.

If the Actor is assigned a forward (e.g. walking) Animation, it will never meet the target Point, because it will always turn so that he is back to the target Point. This command should be used with backward (e.g. backward walking) Animations only.

When the Actor reaches the target Point it will not stop by itself, but will walk straight infinitely (in the last direction he aimed). To make it stop a static Animation must be assigned to it again.

GOTO_SYM_POINT command pauses execution of the script until Actor reaches the desired Point. Thus it is not necessary to use any waiting commands, as for door commands. In the following example the ANIM 0 line will be executed only after the Actor reaches the Point 1.

Example:
LABEL 1
ANIM 2 //assign backward walking animation
GOTO_SYM_POINT 1
ANIM 0 //assign standing animation
STOP
END

The same additional considerations apply here as for GOTO_POINT command. It is recommended to read them as well.
