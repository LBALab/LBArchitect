Track Script: STOP (control statement)

STOP breaks current Track execution in the Track Script.

Syntax:
  • STOP

STOP is necessary to split separate Tracks from each other. Without it the Script would execute without break all the way to the end. In the example below first STOP prevents the Script from going to LABEL 11 after executing LABEL 10. After STOP has been reached the Script is halted and can be run again only by calling SET_TRACK(_OBJ) Life Script command.

Example:
label 10
sample 28
stop
label 11
sample 23
stop
