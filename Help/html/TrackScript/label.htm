Track Script: LABEL (control statement)

LABEL defines a Track in the Track Script.

Syntax:
  • LABEL id(0 to 255 integer) [name(text)]
Parameters:
  • id is the reference number of the Track.
  • name (optional) is the reference name of the Track (LBArchitect only).

LABEL defines a Track, i.e. a sequence of commands that can be called from within the same Track Script (GOTO statement) or from Life Scripts of all Actors (SET_TRACK(_OBJ) command). Usually such a sequence starts with the LABEL statement and ends with the STOP statement.

Current Track can be tested with the CURRENT_TRACK Life Script variable.

It is necessary to always define the label id, because its value is stored in the binary form of the Script. Also, CURRENT_TRACK value can be compared using less than (<) and greater than (>) operators, not only equal (==) operator.
