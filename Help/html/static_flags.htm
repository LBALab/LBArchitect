Actor Static Flags

Static flags set various Actor boolean properties:

Can collide with objects

This flag changes the way the Actor interacts with other Actors. If this flag is set, the Actor will behave like a real wold object, colliding with other Actors. If the flag is not set, the Actor will push other Actors if it walks into them, like they didn't exist. Also he won't be able to attack other Actors (won't give any damage). If two Actors have the flag not set, they will be able to walk through each other like ghosts.

This flag can be changed at run time using the OBJ_COL Life Script command.

Can collide with Bricks

If this flag is set, the Actor will be bound by the world physics, like gravitation (will fall if not standing on the ground), and walls will be solid to it. While this flag is not set, the Actor will be able to walk though walls, walk in the air, won't fall, and he won't be able to jump on higher ground levels.

This flag can be changed at run time using the BRICK_COL Life Script command.

Can detect Zones

If the flag is set, the Actor will be able to detect if it's inside a Sceneric Zone or not. Otherwise the Life Script ZONE variable will always report that the Actor is not inside any Zone.

Example:
if zone == 2
explode_obj 1
else
message 20
endif
In the above example, if the flag is not set the message 20 line will always be executed instead of explode_obj 1 line, even if the Actor is inside the Zone 2.

Uses clipping

This flag works for Sprite Actors only. If this flag is set, the Actor will be invisible (clipped) outside the clipping rectangle. This flag is used to make the doors slide "into" the walls and not appear on the other side (if the wall is thinner than the door's width).

Can be pushed

If the flag is set, other Actors will push the Actor if it stands on their way. This flag should be used for Sprites only. If it is used for a 3D Actor, it will work rather strangely depending on the Can collide with objects flag.

Low collision

Unknown.

Can test floor type

If the flag is set, the Actor will die if he walks into the water. Otherwise it will be able to walk over the water as it was a solid floor.

Hidden

If the flag is set, the Actor will not be present in the world.

This flag can be changed at run-time using the INVISIBLE Life Script command. An invisible Actor is still an obstacle, and plays animations and sounds (though you can't see the animations :)).

Is Sprite

If the flag is set, the Actor will be a Sprite Actor. Sprite Actors do not have Bodies or Animations, but static images (sprites) instead. Their dimentions in isometric 3D Scene are described in entry 3 (counting from 0) of the ress.hqr file.

Sprite Actors have special commands to open and close them if they seve purpose of doors, set their speed, and move them.

Can fall

If the flag is set, the Actor will be able to fall if he's not standing on the ground. For this flag to work, Can collide with Bricks flag must be also set.

This flag can be changed at run-time using the CAN_FALL Life Script command.

Doesn't cast shadow

If the flag is not set the Actor will cast shadow: a small semi-transparent gray circle on the ground right below the Actor. The shadow circle is always the same size (about one Brick in width and height).

Backgrounded

If this flag is set, the Actor will not be redrawn all the time, but only when the screen is refreshed by an other event (another Actor passing by, conversation text drawing, etc.) For example if the flag is set for an Actor, and the Actor is moving, its movement won't be shown. It will stay still at the same place all the time. Only when the old place is redrawn because of another event, it will disappear from there, and when the place where it is now is redrawn by another event, the Actor will be drawn too.

This flag can be changed at run-time using the BACKGROUND Track Script command.

Carrier

If the flag is set, the Actor will be able to carry another actor on its top, otherwise no other Actor will be able to stand on top of it.

Uses Mini ZV

"ZV" stands for Zone Volume. If the Flag is set, the Actor's bounding box dimensions in X and Z (horizontal) axes will be set to the smaller Actor dimension of the X and Z axes. If the Flag is unset, average of the X and Z dimensions will be taken instead. This Flag does not influence Y (vertical) axis.

This flag applies to 3D Actors only.
