Life Script: ZONE (variable)
Life Script: ZONE_OBJ (variable)

ZONE returns Scene Index of the Zone that current Actor is inside at the moment, ZONE_OBJ returns Zone index for the Actor specified in the parameter.

Syntax:
  • if ZONE == index(0 to 255 integer)
  • if ZONE_OBJ actor(0 to 255 integer) == index(0 to 255 integer)

(if may be also: or_if, swif, oneif; == may be also: !=, >, <, >=, <=)

Parameters:
  • actor is the index of the target Actor (the Actor must exist in the Scene).
Return value:
  • ZONE returns virtual (Scene) Index of the Zone that the current Actor is currently in,
  • ZONE_OBJ returns virtual (Scene) Index of the Zone that the specified Actor is currently in.

Scene Index of a Zone is a special property of Sceneric Zones. Other types of Zones do not have it, so it is not possible to check presence of an Actor inside them. The Scene Index for each Sceneric Zone is configured manually and it is not related to index of the Zone in the Zones array. More than one Sceneric Zone may have the same Scene Index. In this case a conditional statement like ZONE == 3 will be met if the Actor is inside any of these Zones.

If the Actor is not in any Sceneric Zone, ZONE will return -1, but this case cannot be tested. Although the Zone index value is one byte, so 255 should be equal to -1, the engine keeps the value as a 4-byte (dword) integer internally, and -1 as dword (232) is not equal to -1 as byte (255). Thus if the Actor is not in any Sceneric Zone, the condition will be false for any comparison value.

Examples:
if zone == 1 //if Actor is inside Sceneric Zone 1
//perform operations
endif

if zone_obj 5 > 8 //if Actor 5 is inside any Sceneric Zone whose Sceneric Index is greater than 5
//perform operations
endif
