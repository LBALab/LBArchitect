Life Script: COL (variable)
Life Script: COL_OBJ (variable)

COL returns id of an Actor that collides with the current Actor, COL_OBJ returns id of an Actor the collides with specified Actor.

Syntax:
  • if COL == index(0 to 255 integer)
  • if COL_OBJ actor(0 to 255 integer) == index(0 to 255 integer)

(if may be also: or_if, swif, oneif; == may be also: !=, >, <, >=, <=)

Parameters:
  • actor is the index of the target Actor (the Actor must exist in the Scene).
Return value:
  • COL returns index of the Actor that is colliding with current Actor,
  • COL_OBJ returns index of the Actor that is colliding with specified Actor.

An Actor collide with another Actor if at least one of them is trying to move towards the other, and cannot, because it blocks its way. On the other hand, when two Actors are just standing next to each other (touching), it is not considered a collision.

If the Actor is not colliding with any other Actor, COL(_OBJ) will return -1, but this case cannot be tested. Although the Actor index value is one byte, so 255 should be equal to -1, the engine keeps the value as a 4-byte (dword) integer internally, and -1 as dword (232) is not equal to -1 as byte (255). Thus if the Actor does not collide with any Actor, the condition will be false for any comparison value.

Examples:
if col == 1 //if current Actor collides with Actor 1
//perform operations
endif

if col_obj 5 == 1 //if Actor 5 collides with Actor 1
//perform operations
endif
