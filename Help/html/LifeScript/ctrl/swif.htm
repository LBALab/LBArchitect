Life Script: SWIF (control statement)

SWIF begins conditional statements, that allow executing particular piece of code only if some conditions are met. It works just like IF, but will repeat execution only if the condition result has changed.

Syntax:
  • SWIF variable [actor(0 to 255 integer)] operator value
Parameters:
  • variable is name of the world property to be evaluated. For list of variables see Life Script overview.
  • actor is index of the Actor to test the variable for. This parameter is required only for some variables (mostly the ones that end with _OBJ).
  • operator can be one of the following:
    ==equal
    !=not equal
    >greater than
    <less than
    >=greater or equal
    <=less or equal
  • value is the value to compare the variable with. Data type of the value depends on the variable used. The value must be constant, it cannot be a variable.

Conditional statements allow for testing variables of the world against given values and executing specified Script pieces depending whether the condition is met or not. The end of the Script block must end with ENDIF statement, even if it consists of a single line only. SWIF checks one condition and executes associated Script block if the condition is met, but executes it only once. Next execution may be performed only if the condition has been "noticed" in a false (not met) state, and then turned true again. The Script block will be executed only once per each false->true condition cycle. SWIF should not be nested in other SWIF statements, because in most cases the Script block will never execute. However it may be useful for some unusual solutions. OR_IF statement cannot be used with SWIF (it will allow execution of the block more than once per cycle).

In the example below the message 23 will be shown once when the Actor enters Zone 1, then it may be executed again only if the Actor leaves the Zone 1 and enters it again.

ELSE statement will be executed always when the block under condition is not executed. This means: when the condition is false and also when the condition is true, but the 'true' Script block had been executed before (and has not changed since).

Example:
swif zone == 1
message 23
endif
